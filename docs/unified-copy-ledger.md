# UnifiedVideoDownloader Copy Ledger

This ledger tracks migration of the 81 `UnifiedVideoDownloader` features into the WXT/React/TypeScript target. It is a planning and audit document: a feature is not considered copied until the target contract, fixtures, tests, policy review, UI wiring where applicable, and build verification are complete.

## Status Legend

- `implemented`: Ported or redesigned in the target with typed contracts and tests.
- `already-present`: Current target source already has the baseline feature.
- `partial`: Current target has a shell or narrower implementation that needs parity work.
- `port-core`: Port or redesign core/background/protocol/storage behavior.
- `port-ui`: Port or redesign user-facing React controls and state wiring.
- `policy-only`: Keep detection, warning, restriction, or documentation behavior only; do not implement bypass or credential extraction.
- `defer`: Do not implement until safe fixtures, product scope, and policy constraints are clear.
- `intentionally-deferred`: Final audit state for features left out because no safe fixture or product-safe behavior exists.

## Dependency ID Prefixes

- `BOOT`: Background bootstrap, runtime APIs, settings services, permissions, and startup orchestration.
- `CAND`: Candidate detection, normalization, dedupe, enrichment, and media evidence.
- `POL`: Policy, protected-media, restriction, ToS, geo, and provider authorization boundaries.
- `PROTO`: Direct, HLS, DASH, parser, planner, and protocol selection behavior.
- `DL`: Download queue, segment scheduler, resume, export, mux, notifications, and context menu behavior.
- `UI`: Side panel, popup, history, cards, selectors, preview, trim, and progress surfaces.
- `PLUGIN`: Site-specific and streaming-host detector framework, registry, and safe fixture-backed plugins.
- `E2E`: End-to-end fixtures, smoke tests, and parity matrix verification.
